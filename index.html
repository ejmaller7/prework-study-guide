HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Semantic elements describe the roles the body element has in a webpage
  • Metadata: data about our webpage that the browser uses to effectively server our webpage to users
  • Parent/child element: Often used by developers to explain the relationship among elements in HTML

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are 3 main ways to style a webpage using CSS: Inline CSS, Internal CSS Style Sheet & External CSS Style Sheet.
  • Inline CSS: Styles an element directly in the HTML file.
  • Internal CSS style sheet: Contains CSS rules for the webpage in the head of the section of the HTML file.
  • External CSS style sheet: Allows developers to keep all their CSS rules in a separate file, which makes design changes easier.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout main: switches back to the main branch
  • git commit: takes the staged snapshot and commits it to the project history
  • git pull/push: pulls a branch form a remote repository into the current branch/pushes a local branch to another repository

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Different types of variables: String, Number, Boolean, Array, Object.
  • A function is a set of instructions that tells the computer how to perform a certain task
  • Functions do not automatically execute when the JavaScript file is run, unlike a for loop or conditional statement.
